编译原理-中间代码,代码生成

发表于 2020-01-04 分类于 编译原理 阅读次数： Valine：

一些概念

  • 用语法制导定义和翻译方案的方法将源程序翻译成中间形式
  • 中间代码特点
    • 介于源语言与目标代码之间
    • 区分前端与后端, 方便提出针对新机的编译器
    • 可以设计针对中间代码的优化器
    • 便于移植
  • 常用中间形式
    • 后缀形式
    • 图形表示
    • 三地址代码

后缀表示

对于表达式E

  • E为变量或常数, 其后缀表示为其本身
  • E为E1 op E2, 则后缀表示为(E1后缀表示) (E2后缀表示) op
  • 后缀表示不需要括号
  • 优点: 便于计算机处理表达式

图形表示

  • 语法树是一种图形表示
  • 有向无环图也是一种中间表示

三地址代码

  • 一般为 x:= y op z
  • 如 x + y z, 三地址代码为 t1 := y z, t2 := x + t1
  • 赋值 x:= y op z, x := op y, x := y
  • 无条件转移 goto L
  • 条件转移 if x reloy y goto L
  • 过程调用 param x 和 call p,n
  • 过程返回 return y
  • 索引赋值 x:=y[i] 和 x[i]:=y
  • 地址, 指针赋值: x:=&y, x:=y, x:=y

声明语句

  • 为局部名字建立符号表条目
  • 为其分配存储空间
  • 符号表中包含名字的类型和分配给它的存储单元的相对地址等信息

作用域信息的保存

  • 符号表的特点
    • 各过程有各自的符号表
    • 符号表之间有双向链
    • 构造符号表时需要符号表栈
    • 构造符号表需要活动记录栈
