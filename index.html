编译原理-中间代码,代码生成

发表于 2020-01-04 分类于 编译原理 Valine：

一些概念

  • 用语法制导定义和翻译方案的方法将源程序翻译成中间形式
  • 中间代码特点
    • 介于源语言与目标代码之间
    • 区分前端与后端, 方便提出针对新机的编译器
    • 可以设计针对中间代码的优化器
    • 便于移植
  • 常用中间形式
    • 后缀形式
    • 图形表示
    • 三地址代码

后缀表示

对于表达式E

  • E为变量或常数, 其后缀表示为其本身
  • E为E1 op E2, 则后缀表示为(E1后缀表示) (E2后缀表示) op
  • 后缀表示不需要括号
  • 优点: 便于计算机处理表达式

编译原理-运行时存储空间的组织与管理

发表于 2020-01-04 更新于 2020-01-05 分类于 编译原理 Valine：

一些概念

  • 通常的命令式编程语言表现为多个函数的实现与相互调用
    • 入口函数main
    • 代码执行过程, 实际是依据过程调用关系形成一格调用结构图
    • 每个过程的执行需要一定的辅助空间来辅助
  • 程序的执行过程中, 程序中数据的存取是通过与之对应的存储单元进行的
  • 程序使用的存储单元是由标识符表示的, 其对应的内存地址是由编译程序在编译时由其生成的目标程序运行时分配
  • 过程的一次执行被称为过程的一次活动(过程可以理解为”函数”, 而活动则可以视为”函数的调用”)
  • 活动记录: 活动过程中用于存放所需信息的存储空间
  • 过程P的一次活动的生存期, 指的是从执行该活动期的第一步操作到最后一步操作之间的操作时间, 包括执行P式调用其他进程所花费的时间
    • 过程递归时, 生存期指最外层, 即从开始到结束所有时间
    • 一个过程可以对应多个活动, 每次调用即一次活动

编译原理-语义分析-语法制导的翻译

发表于 2019-12-10 更新于 2020-01-04 分类于 编译原理 Valine：

语法制导

定义

  • 对应每一个产生式编制一个语义子程序, 当一个产生式获得匹配时, 调用相应的语义子程序实现语义检查与翻译
  • 带属性和规则的上下文无关文法, 称之为基础文法

翻译方案

在产生式的右部的适当位置, 插入相应的语义动作, 按照分析的进程, 执行遇到的语义动作

属性文法-上下文无关文法与程序语义的桥梁

  • 亦称为翻译文法
  • 在上下文无关文法的基础上, 为每个文法符号配备若干相关的”值”(属性)
    • 属性代表与文法相关的信息
    • 属性可以进行运算与传递
    • 语义规则: 对于文法的每个产生式都配备了一组属性的计算规则

编译原理-文法分析-自下而上分析

发表于 2019-12-08 更新于 2020-01-03 分类于 编译原理 Valine：

一些概念

规约: 对应最右推导的逆过程

句柄: 和某产生式右部匹配的子串

  • 句柄是句型的一个字串
  • 把句柄规约为非终结符代表了某一步最右推导的逆过程
  • 句柄的右边只有终结符
  • 若文法二义, 则句柄不唯一

移进-规约分析

  • 输入: 以$结尾的句子
  • 栈: 栈底为$的栈
  • 分析表: 确定下一步动作为移进还是规约
  • 分析程序
    • 四种动作
      • 移进: 下一个输入符号压栈
      • 规约: 确定使用那个产生式
      • 接受: 输出分析成功
      • 报错: 发生语法错误, 调用错误恢复例程
    • 对于每个字符, 若可规约则规约, 不可规约则继续移进判断可否规约(查表)
  • 移进-规约分析的冲突
    • 要移进还是规约?
  • 规约-规约冲突
    • 选择那个产生式进行规约?

LL(1)文法

先定义两个函数:

FIRST集

  • FIRST(α) = {a | α $\sf ⇒^*$ a…, a $\sf\in V_T$}

    • FIRST(α)是从α推导出的串的起始终结符的集合
    • 特别的是: α $\sf ⇒^* $ε 时, 规定 ε $\sf \in$ FIRST(α)
    • 对于A的任何两个选择$\alpha_i$和$\alpha_j$, 希望有FIRST($\alpha_i$)⋂FIRST($\alpha_j$) = ∅
      但是需要FIRST($\alpha_i$)和FIRST($\alpha_j$)都不含ε

    • FIRST集合计算方法:

      • 若X⟶a… 则将终结符a加入FIRST(X)中
      • 若X⟶ε, 则将ε加入FIRST(X)中
      • 若X⟶Y…, 且Y为非终结符, 则将FIRST(Y)加入FIRST(X)中, 若FIRST(Y)中有空, 则将FIRST(Y)去掉ε加入FIRST(X)中, 在看下一个非终结符

      • 例子:

          S ⟶ BA
  A ⟶ BS|d
  B ⟶ aA|bS|c
  则
  FIRST(S) = FIRST(B) 
  FIRST(A) = FIRST(B) ⋃ {d}
  FIRST(B) = {a,b,c}
